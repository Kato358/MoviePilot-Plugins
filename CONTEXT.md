# CONTEXT.md — P115StrgmSub 领域术语表

> 本文件记录项目核心领域概念，不含实现细节。

---

## 核心概念

**订阅 (Subscribe)**: MoviePilot 中用户创建的媒体追更请求。包含媒体名称、类型（电影/剧集）、季号、缺失集数等。订阅有状态：N(进行中)、R(已暂停)、F(已完成)。

**搜索源 (Search Source)**: 用于查找 115 网盘资源的外部服务。搜索源按优先级回退：当前源无结果时自动尝试下一个。

**转存 (Transfer)**: 将 115 网盘分享链接中的文件保存到用户个人网盘目录的操作。

**洗版 (Best Version)**: 订阅的高级模式。启用时即使已有资源，也会继续搜索更高质量的资源进行替换。通过评分机制判断"更优"。

**屏蔽系统订阅 (Block System Subscribe)**: 插件的核心功能之一。阻止 MoviePilot 默认的订阅处理流程，仅使用本插件的 115 网盘转存。支持窗口期机制：在最后一次任务完成后临时恢复系统订阅，窗口到期后重新屏蔽。

**频道 (Channel)**: Telegram 频道，资源分享的源头。每个频道有特定的帖子格式和资源类型。

## 搜索与匹配

**关键词降级 (Keyword Fallback)**: 搜索时先用精确关键词（如"权力的游戏1"），无结果时降级到模糊关键词（如"权力的游戏"）。

**SubscribeFilter**: 订阅过滤条件（质量/分辨率/特效），用于在搜索结果中筛选符合要求的资源。严格模式下不匹配直接跳过，洗版模式下不匹配但给予低分。

**资源评分 (Resource Score)**: 基于 SubscribeFilter 的匹配度打分。每项匹配（质量/分辨率/特效）+100 分。用于洗版模式判断是否需要替换已有资源。

## Telegram 集成

**HTTP 模式**: 零配置模式。通过 `t.me/s/<channel_name>` 抓取公开频道消息，增量存储到本地 SQLite，搜索时查询本地数据库。

**MTProto 模式**: 增强模式。通过 Telethon 库直接连接 Telegram，支持服务端关键词搜索、搜索全部历史消息。首次使用需扫码登录。

**消息解析 (Message Parser)**: 从 Telegram 频道帖子中提取 115 网盘链接和资源标题的逻辑。针对不同频道格式有不同的解析策略。

## 外部服务

**115 网盘**: 中国网盘服务，插件的核心转存目标。有严格的 API 风控机制（速率限制、批量限制）。

**MoviePilot**: 媒体管理平台，提供订阅管理、媒体识别（TMDB）、下载历史等功能框架。
